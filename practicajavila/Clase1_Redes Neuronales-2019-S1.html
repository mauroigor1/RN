Las redes neuronales artificiales son una herramienta poderosa de Machine Learning. Pero antes de entrar en el campo de las redes neuronales, es importante saber tambin que dentro del Machine Learning, hay tres tipos fundamentales de aprendizaje:

  • Aprendizaje Supervisado / Supervised Learning
  • Aprendizaje No Supervisado / Unsupervised Learning
  • Aprendizaje Reforzado / Reinforcement Learning

El Aprendizaje Supervisado contempla aquellos problemas en los cuales cada observacin de un cierto fenmeno cuenta con las variables de entrada y tambin de salida. Es decir, conocemos cules son las caractersticas de cierto fenmeno y tambin cul debiera ser la respuesta esperada. Un ejemplo de esto puede ser la prediccin del dolar en base a variables como los cambios en la bolsa, o tambin la prediccin de la temperatura de un da en particular en base a la estacin del ao, la temperatura del da anterior, y otras variables. Es este tipo de aprendizaje es donde se concentra la mayor parte de las Redes Neuronales Artificiales.

En el Aprendizaje No Supervisado se cuentan con datos, pero no se tiene una variable de salida definida. El objetivo de atacar este tipo de problemas es encontrar caractersticas en comn entre los datos, comnmente, hacer anlisis de Clusters y con ello, aplicar algoritmos de recomendacin. Por ejemplo, dadas las puntuaciones a series de Netflix, recibir recomendaciones de qu me puede gustar en base a los gustos de otros individuos que tienen gustos similares a los mios (lo mismo aplica a youtube). Hablando en trminos de redes neuronales, tenemos las Redes de Kohonen.

Conjuntos de entrenamiento, validación y testeo.¶

En el contexto del aprendizaje supervisado tenemos la capacidad de evaluar y cuantificar, bajo ciertas mtricas, el desempeo de nuestro modelo. Por supuesto que esto aplica directamente a redes neuronales.

En estos problemas nuestro objetivo es entrenar un modelo que sea capaz de predecir/clasificar algn fenmeno en funcin de cierta informacin. El proceso se separa en dos:

  1. Etapa de Entrenamiento: usualemente el 70% u 80% de nuestros datos totales se destina a la etapa de entrenamiento. Estos datos son nuevamente separados en dos conjuntos disjuntos:

    • Conjunto de Entrenamiento: corresponde al conjunto de datos con el que se entrenar el modelo.
    • Conjunto de Validacin: corresponde a un pequeo conjunto de datos que, en cada poca del entrenamiento se evala sobre el modelo. Esto nos dar una idea cercana sobre qu performance tendr el modelo sobre los datos de entrenamiento. Tambin nos servir para analizar si nuestro modelo est aprendiendo bien, se est sobre ajustando (overfitting), o est aprendiendo de muy mala manera (underfitting). Ms adelante veremos cmo utilizar los resultados de este conjunto de validacin.

  2. Etapa de Testeo: los datos restantes luego de elegir los de entrenamiento (30% o 20%). Son utilizados para evaluar el desempeo del modelo entrenado sobre datos que jams ha visto antes. Si asumimos que nuestros datos son consistentes y los datos de entrenamiento contienen suficientes datos para abarcar la variabilidad de los datos existentes, entonces deberamos obtener resultados de desempeo similares a los obtenidos en el entrenamiento

Funciones de activación¶

Algunas de las funciones de activacin ms usuales son:

1) Función Sigmoidal¶

\begin{equation} g(z) = \dfrac{1}{1+e^{-z}} \end{equation}

2) Función Tangente Hiperbólica¶

\begin{equation} g(z) = \dfrac{e^z - e^{-z}}{e^z+e^{-z}} \end{equation}

3) Función ReLU¶

\begin{equation} g(z) = \max(0,z) \end{equation}

4) Funcion Leaky ReLU¶

\begin{equation} g(z) = \max(\varepsilon z,z) , \varepsilon << 1 \end{equation}

# Definir la funcin sigmoidal
def sigmoid(z):
    return(1/(1+ np.exp(1)**-z))

# Funcin Tangente Hiperblica
def tanh(z):
    return (np.exp(1) - np.exp(1)**-z) / (np.exp(1) + np.exp(1)**-z) 

# Funcin ReLU
def relu(z): 
    return max(0.0,z)

# Funcin Leaky ReLU
def leakyrelu(z, eps=0.1):
    'Por defecto se defini un eps = 0.01'
    return max(eps*z,z)

Ejemplos de esto son la estandarizacin al intervalo $[0,1]$, $[-1,1]$ o una normalizacin del tipo $\dfrac{x-\mu}{\sigma}$, donde $\mu$ es el promedio de las observaciones de una variable especfica y $\sigma$ su desviacin estndar.


Escalamientos clásicos para un dato¶
Supongamos que un dato $x$ de una muestra $X = {x_1,...,x_n}$ lo queremos reescalar a un valor $z$. Conocemos $\mu$ (el promedio de la muestra $X$), $\sigma$ (su desviacin estndar), $\max$ y $\min$ de $X$.


Al intervalo $[0,1]$¶
Note que aqu necesitamos encontrar una funcin $f: Dom(f) \subset \mathbb{R} \rightarrow [0,1]$
donde $Dom(f) = [\min, \max]$.


Teniendo lo anterior en consideracin, es sencillo observar que la funcin buscada para reescalar nuestros datos al intervalo $[0,1]$ est dada por:


$$f(x) = \dfrac{x - \max}{\max - \min} $$


Al intervalo $[-1,1]$¶
Ahora, si multiplicamos la funcin anterior por $2$, obtendremos resultados entre $[0,2]$, luego basta trasladar la funcin en $1$ para obtener resultados en el intervalo $[-1,1]$.
$$f = 2\dfrac{x - \max}{\max - \min} - 1$$


A un intervalo $[a,b]$ cualquiera¶
Y ahora puede notar que en realidad el caso ms general, donde $Rec(f)=[a,b]$ nos lleva a definir un escalamiento de la forma: 
$f(x) = (b-a)\dfrac{x - \max}{\max - \min} + a$


Normalizar como una $\mathcal{N}(0,1)$¶
Otro escalamiento clsico es normalizar los datos, es decir, obligarlos a que el promedio de los datos transformados sea igual a 0, y que su desviacin estndar sea 1. Esto har que la mayora de los datos viva alrededor del 0, variando entre $-1$ y $1$ (pero ojo, la mayora, no todos).


$$z = \dfrac{x-\mu}{\sigma}$$

Hay alguna manera de usar el overfitting a mi favor?
El usar demasiadas pocas tiene la desventaja de que es altamente probable entrar en overfitting, pues los datos pasarn tantas veces por la red que esta tender a memorizarlos, pero si nuestro modelo se fue entrenando bien hasta cierta poca, podramos querer quedarnos con el mejor modelo que se haya entrenado (luego de cada poca tenemos un modelo distinto, pues son nuevos pesos los que se actualizan).


Afortunadamente tenemos una forma sencilla de hacer esto. Es una funcionalidad de keras llamada checkpoint, la cual consiste en guardar los pesos despus de cada poca, siempre y cuando el valor loss_val (prdida en el conjunto de validacin) decrezca. Esto se aplica previo a entrenar nuestro modelo. Lo veremos en un ejemplo ms adelante.
